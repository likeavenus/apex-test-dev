# APEXGARD — инструкция: свой сервер в Selectel

Сайт будет лежать **на вашем сервере**, открываться по **apexgard.ru**.  
GitHub и аккаунт разработчика для хостинга **не используются**.

---

## Что нужно купить

### 1. Облачный сервер (VPS) в Selectel

1. Войдите в [my.selectel.ru](https://my.selectel.ru)
2. **Облачная платформа** → **Серверы** → **Создать сервер**
3. Параметры (достаточно для лендинга):

| Параметр | Рекомендация |
|----------|--------------|
| ОС | **Ubuntu 24.04 LTS** (или 22.04) |
| CPU | 1 vCPU |
| RAM | 1 GB |
| Диск | 10 GB SSD |
| Регион | любой (Москва / SPB — как удобнее) |

Этого хватит с запасом для статического сайта + HTTPS.

**Не нужно:** Kubernetes, managed DB, балансировщик, Docker-облако — только обычный VPS.

4. Создайте сервер и **запишите публичный IP-адрес** (например `185.xxx.xxx.xxx`).

---

### 2. DNS для домена apexgard.ru

Домен у вас уже в Selectel — привяжите его к серверу.

**Домены** → **apexgard.ru** → **DNS-записи**:

| Тип | Имя | Значение |
|-----|-----|----------|
| A | `@` | **IP вашего сервера** |
| A | `www` | **IP вашего сервера** |

Удалите старые A/CNAME, если они вели на заглушку или GitHub.

Проверка (через 15–60 мин): в браузере по IP или домену — пока может быть пусто, это нормально до деплоя.

---

### 3. SSH-доступ для разработчика

Разработчику нужен доступ **только к этому серверу**, не ко всему аккаунту Selectel.

**Вариант А (проще):** создать пользователя для деплоя

1. На сервере (через консоль Selectel «VNC / консоль», если SSH ещё не настроен) или попросите разработчика после первого входа:
   - пользователь `deploy` с правами sudo **или** доступ только к `/var/www/apexgard`
2. Передать разработчику:
   - **IP сервера**
   - **логин** (например `root` или `deploy`)
   - **SSH-ключ** (разработчик пришлёт свой публичный ключ — `.pub` файл) **или** временный пароль (потом сменить)

**Вариант Б:** добавить SSH-ключ разработчика в панели Selectel при создании сервера (если есть такая опция).

## Что передать разработчику (чеклист)

Скопируйте и заполните:

```
Сервер APEXGARD — данные для деплоя

IP сервера:     ___________________
ОС:             Ubuntu 24.04
SSH логин:      ___________________
SSH доступ:     ключ добавлен / пароль в личке

Домен:          apexgard.ru
DNS:            A @ и www → IP сервера (готово: да / нет)

Яндекс Метрика: код счётчика (когда будет): ___________________
Instagram:      https://www.instagram.com/tonys.ws (уже на сайте)
```

---

## Что сделает разработчик после этого

1. Подключится по SSH
2. Установит nginx + SSL (HTTPS, бесплатный сертификат)
3. Зальёт файлы сайта
4. Проверит https://apexgard.ru и https://www.apexgard.ru
5. Отдаст QR-код и подключит Метрику (когда пришлёте код)

## Частые вопросы

**Нужен ли GitHub заказчику?**  
Нет. Сайт живёт на вашем сервере.

**Нужно ли отдавать пароль от Selectel?**  
Нет. Достаточно SSH на **один** сервер (лучше отдельный пользователь, не root).

**Можно ли потом сменить разработчика?**  
Да. Сервер и домен ваши — новый специалист получит те же SSH-доступы.

**Сайт упадёт, если разработчик удалит репозиторий у себя?**  
Нет. На VPS копия сайта независима.
